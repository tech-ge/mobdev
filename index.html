Step 1: Variables & Data Types

What are Variables?

Variables are like containers that store information in your program. Think of them as labeled boxes where you can put different things.

var name = "John"; // Stores text (String)
var age = 25; // Stores a number (Integer)
var isStudent = true; // Stores true/false (Boolean)

In programming, we use variables to store data that can be used and changed later in our code.

Why Use Semicolons and Commas?

Semicolons (;) tell the computer where one instruction ends and another begins. Commas (,) are used to separate items in a list.

// Semicolons end statements
var firstName = "Jane";
var lastName = "Doe";

// Commas separate items in a list
var colors = ["red", "green", "blue"];

Step 2: Functions

What are Functions?

Functions are reusable blocks of code that perform a specific task. They help you avoid writing the same code multiple times.

// Defining a function
function greet() {
  return "Hello, world!";
}

// Using (calling) the function
var message = greet(); // message now contains "Hello, world!"

Function Parameters

Functions can accept inputs called parameters, which make them more flexible.

// Function with parameters
function greetPerson(name) {
  return "Hello, " + name + "!";
}

// Using the function with different inputs
var greeting1 = greetPerson("Alice");
var greeting2 = greetPerson("Bob");
Tip: The parentheses () after the function name are where you put the parameters. Curly braces {} contain the code that runs when the function is called.

Step 3: Control Flow

Making Decisions with If/Else

Control flow lets your program make decisions and execute different code based on conditions.

var temperature = 75;

if (temperature > 80) {
  console.log("It's hot outside!");
} else if (temperature > 60) {
  console.log("It's pleasant outside.");
} else {
  console.log("It's cold outside!");
}

Loops: Repeating Actions

Loops let you repeat code multiple times, which is useful for working with lists of items.

// For loop - repeats a specific number of times
for (var i = 0; i < 5; i++) {
  console.log("Count: " + i);
}

// While loop - repeats while a condition is true
var count = 0;
while (count < 3) {
  console.log("Hello!");
  count++; // Increase count by 1
}
Tip: The symbols < (less than), > (greater than), <= (less than or equal to), and >= (greater than or equal to) are comparison operators used in conditions.

Step 4: Objects & Classes

What are Objects?

Objects let you group related data and functions together. Think of them as containers that hold both properties (data) and methods (functions).

// Creating an object
var person = {
  firstName: "John",
  lastName: "Doe",
  age: 30,
  getFullName: function() {
    return this.firstName + " " + this.lastName;
  }
};

// Using the object
console.log(person.firstName); // "John"
console.log(person.getFullName()); // "John Doe"

Classes: Blueprints for Objects

Classes are templates for creating multiple objects with the same structure.

// Defining a class
class Person {
  constructor(name, age) {
    this.name = name;
    this.age = age;
  }

  introduce() {
    return "Hi, I'm " + this.name + " and I'm " + this.age + " years old.";
  }
}

// Creating objects from the class
var person1 = new Person("Alice", 25);
var person2 = new Person("Bob", 30);
Tip: The this keyword refers to the current object. In a class, this.name means "the name property of this specific object".

Step 5: Build Your First App

Putting It All Together

Now let's use everything you've learned to create a simple mobile app - a task manager!

// Task class to represent each task
class Task {
  constructor(name, completed = false) {
    this.name = name;
    this.completed = completed;
  }
  
  toggleComplete() {
    this.completed = !this.completed;
  }
}

// TaskManager class to manage all tasks
class TaskManager {
  constructor() {
    this.tasks = [];
  }
  
  addTask(name) {
    var newTask = new Task(name);
    this.tasks.push(newTask);
  }
  
  getTasks() {
    return this.tasks;
  }
}
